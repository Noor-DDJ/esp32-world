## 📖 项目简介

这是一个单文件、零依赖、可分享的网页作品，把乐鑫官网中分散的 ESP32 应用案例，编织成一张可以拖拽探索的世界地图。
灵感启发是：https://flipbook-page.com

用户可以：

- 漫游探索:8 大行业场景平台
- 下钻查看:每个场景下的真实应用设备
- 追溯依据:所有产品卖点均带 `[来源]` 标注，可在乐鑫官方文档中核实

## 🎯 解决的问题

乐鑫的产品技术资料极其丰富，但**应用场景的呈现是分散的**：

- 应用案例散布在产品页、开发板用户指南、官方博客、GitHub 等多个入口
- 非工程师用户（决策者、媒体、新开发者）难以快速建立整体认知
- 缺少一个**可分享、可漫游、视觉化**的场景索引

本项目尝试填补这个空缺——用一个交互式可视化作品，把"芯片—框架—场景—案例"四要素整合到一个页面里。

## 💡 设计理念

### 三条核心原则

1. **每一条卖点都要能在乐鑫官方资料中找到出处** —— 对内容的尊重
2. **每一个交互都要在 200ms 内有反馈** —— 对用户注意力的尊重
3. **30 秒内让一个完全不懂 ESP32 的人理解"乐鑫做什么"** —— 对营销目标的尊重

### 关键设计决策

| 决策 | 备选方案 | 选这个的理由 |
|---|---|---|
| 等距视角世界地图 | 长滚动页 / 标签切换 / 3D | 既有空间感，又能用 2D SVG 高效实现 |
| 8 大场景 | 6 / 10 / 12 | 对应官网"产品应用"主类目，不堆砌 |
| 每场景 3-4 个设备 | 6+ 个 | 讲透"龙头案例"而非堆砌 SKU |
| 沿路径流动的粒子动画 | 静态地图 | 让"设备互联"成为可感知的视觉隐喻 |
| 所有数据 inline 在 HTML | 拆 JSON / 后端 | 单文件零依赖，便于分享演示 |

## 🛠️ 技术实现

```
Frontend:  原生 HTML + JavaScript + SVG
Animation: CSS animation + requestAnimationFrame
Fonts:     Syne (标题) + DM Mono (正文)
依赖:      0 个 npm 包
构建:      不需要
部署:      单文件
```

**关键技术点：**

- **SVG 路径动画**：粒子通过 getPointAtLength() 沿贝塞尔路径运动
- **状态管理**：3 个全局变量覆盖所有 UI 分支
- **性能优化**：拖拽使用 `transform` + `will-change`，保持 60fps
- **响应式**：鼠标 / 触摸事件均已实现
