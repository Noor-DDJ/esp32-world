## 1. 产品概述

ESP32 应用场景百科全书是一个基于 Web 的沉浸式品牌展示页面。用户通过可拖拽的等距视角"世界地图"漫游 8 大行业场景，并可点击查看 ESP32 系列芯片在每个场景下的真实应用设备及其官方依据。

**灵感来源**：https://flipbook-page.com

**简言之**：用一张可交互的世界地图，把乐鑫 ESP32 在各行业的应用案例串联呈现。

---

## 2. 背景与目标

### 2.1 背景

乐鑫现有官网将产品信息按"芯片型号 → 规格 → 应用领域"的层级组织，技术资料完整但场景化呈现分散：

- 应用案例散布在产品页、开发板用户指南、官方博客、GitHub 等多个入口
- 非工程师用户（决策者、媒体、新开发者）难以快速建立"乐鑫覆盖了哪些行业"的整体认知
- 缺少一个**可分享、可漫游、视觉化**的场景索引

### 2.2 目标

- **认知目标**：让用户在 90 秒内建立"ESP32 覆盖 8 大行业、几十种真实设备"的整体印象
- **信任目标**：每条产品卖点均可追溯至乐鑫官方资料，建立信任感
- **传播目标**：单文件可部署、视觉鲜明，便于在社交媒体与销售场景中分发

### 2.3 非目标

| 不做 | 原因 |
|---|---|
| 产品规格对比 | 已有 espressif.com/products 承载 |
| 详细技术文档 | 已有 docs.espressif.com 承载 |
| 直接销售转化 | 承担前置认知任务，不绑定 CTA |
| 通用 IoT 科普 | 内容严格锚定乐鑫官方资料 |

## 4. 信息架构

三层导航结构：

```
Lv0  世界地图（默认视图）
     - 8 个场景平台
     - 道路 + 流光粒子动画
     - 小地图导航 + 拖拽探索

Lv1  场景详情（点击平台进入）
     - 场景背景插画（主题色统一）
     - 3-4 个设备热点（脉冲动画）
     - 右侧面板：设备列表 / 设备详情 切换

Lv2  设备详情（点击热点填充右侧面板）
     - 名称 + 分类 + 一句话描述
     - 4 项关键规格（2×2 网格）
     - 官方依据特性列表（带 [来源] 标注）
     - 选型理由卡片 + 芯片标签
     - 来源说明
```

面包屑导航始终可见：`世界地图 › 场景名 › 设备名`。

### 4.1 八大场景

| 场景 | 主色 | 核心叙事 | 代表设备 |
|---|---|---|---|
| 🏠 智能家居 | 红 | Matter 全屋互联 | 智能灯、Thread 边界路由器、Matter 摄像头、EchoEar 喵伴 |
| 🏢 办公空间 | 蓝 | 安全 + 显示 | LCD 电子价签、支付终端、充电桩通信 |
| 🏭 工厂车间 | 灰 | 工业 AIoT | 预测性维护、机器视觉、Modbus/CAN 网关 |
| 🌾 智慧农业 | 绿 | 极端环境 + 低功耗 | 农田传感、温室控制、农机 CAN |
| 🛍️ 零售商店 | 琥珀 | BLE + 显示 | 电子价签、BLE 信标、移动支付 |
| 🏥 医疗健康 | 蓝灰 | 可穿戴 + 语音 AI | 可穿戴设备、AI 医疗助手、乐鑫×博世感知方案 |
| 📚 教育创客 | 紫 | 开源生态 | 开源开发板、AI 语音玩具、ESP-Drone |
| 🚗 交通出行 | 紫 | 车载 + 充电 | 两轮车仪表盘、充电桩、TWAI/CAN |

### 4.2 设备内容结构（强制规范）

每个设备必须包含 5 个字段：

1. **desc**：一句话说明乐鑫在该场景的角色
2. **specs**：4 项硬规格
3. **features**：3 条卖点，**每条必须带 [来源] 标注**
4. **reason**：选型理由
5. **chips**：3 个核心标签（芯片型号 / 框架 / 协议）

**没有 [来源] 的卖点不允许出现。**

## 5. 功能需求

### 5.1 世界地图（Lv0）

| 功能 | 详细说明 |
|---|---|
| 拖拽探索 | 支持鼠标拖拽与触屏滑动，边界自动 clamp 限制 |
| 平台 hover | 鼠标移入平台时微抬起 + 红色光晕 + 名称气泡 |
| 平台点击 | 进入对应场景详情视图，世界地图淡出 |
| 道路流光 | 5 色道路（红/蓝/绿/橙/紫）+ 沿路径运动的粒子 |
| 小地图 | 右下角缩略图显示视口位置，点击红点直接跳转场景 |
| 装饰粒子 | 30 个浮动小球，慢速上浮淡出，营造氛围 |
| 操作引导 | 底部 instruction bar 说明三个核心操作 |

### 5.2 场景详情（Lv1）

| 功能 | 详细说明 |
|---|---|
| 场景背景 | 主题色径向渐变 + 装饰性 SVG 插画 |
| 设备热点 | 圆形按钮 + 脉冲动画环（2.2s 循环） |
| 热点 hover | 显示设备名称气泡 |
| 热点点击 | 自动切换右侧面板到"设备详情"标签 |
| 设备列表面板 | 列出本场景所有设备，点击切到详情 |
| 标签切换 | "设备列表"与"设备详情"两个标签页 |
| 返回入口 | 左上角"返回世界地图"按钮 + 面包屑 |

### 5.3 设备详情（Lv2）

| 模块 | 详细说明 |
|---|---|
| 标题区 | 设备图标 + 名称 + 分类标签 |
| 描述区 | 一句话说明乐鑫角色 |
| 规格区 | 2×2 网格，4 个关键规格卡片 |
| 官方依据 | 每条特性带红色 [来源] 标注，字号缩小区分 |
| 选型理由 | 红色描边卡片，含理由文字 + 芯片标签 |
| 来源说明 | 底部小字提示所有内容来源 |

## 6. 交互设计要点

### 6.1 核心流程

```
进入页面 → 看到世界全景与流光动画
     ↓
拖拽探索 / 点击小地图跳转
     ↓
点击平台 → 进入场景详情
     ↓
点击热点 OR 列表项 → 查看设备详情
     ↓
点击面包屑 OR 返回按钮 → 回到上一层
```

### 6.2 状态管理

仅维护 3 个全局状态：

- `currentSceneId`（当前场景 id 或 null）
- `currentDeviceId`（当前设备 id 或 null）
- `currentTab`（'devices' 或 'detail'）

### 6.3 响应式要求

- 桌面端：完整体验
- 平板：完整体验，触摸交互
- 移动端：基础可用，横屏体验更佳（v5 优化目标）

## 7. 视觉设计

### 7.1 色板

| 用途 | 色值 |
|---|---|
| 主品牌色（CTA、重点） | `#E8341A` |
| 深底背景 | `#0D0F14` |
| 表面层 | `#161A24` |
| 边框 | `rgba(255,255,255,0.07)` |
| 场景色（红/蓝/绿/橙/紫） | 对应 8 大场景分区 |

### 7.2 字体

| 用途 | 字体 |
|---|---|
| 标题、强调 | Syne（700/800 字重） |
| 正文、数据 | DM Mono（400/500 字重） |

### 7.3 关键视觉元素

- 双层道路（暗色 base + 彩色 glow + 高斯模糊滤镜）
- 沿 SVG 路径运动的粒子流（带拖尾）
- 等距俯视的场景平台（多边形 / 圆角矩形 + 渐变填充）
- 内嵌 SVG 插画（建筑、田野、工厂等场景细节）

## 8. 内容规范

### 8.1 数据来源（强制）

所有内容必须来自以下来源之一：

1. 乐鑫官网（espressif.com 及子页）
2. 芯片数据手册
3. ESP-IDF 官方文档（docs.espressif.com）
4. 乐鑫官方 CSDN 博客
5. 乐鑫官方 GitHub / Gitee 仓库
6. 公开行业案例（须显式标注"非乐鑫官方直接发布"）

### 8.2 内容更新机制

- **场景级**：保持 8 个稳定，避免改版成本
- **设备级**：每季度盘点官方资料，按官方新场景替换
- **来源校验**：每半年抽检 [来源] 链接的可访问性
- **审核流程**：建议由市场部 + 产品技术市场（PMM）共同评审

## 9. 技术实现

### 9.1 技术栈

| 维度 | 选型 | 理由 |
|---|---|---|
| 框架 | 无（原生 HTML + JS + SVG） | 单文件分发，零依赖 |
| 渲染 | SVG（地图、热点）+ HTML（面板） | SVG 适合矢量图形与路径动画 |
| 动画 | CSS animation + requestAnimationFrame | 不引入动画库，控制更精细 |
| 字体 | Google Fonts CDN | 无需打包 |
| 数据 | 内嵌 JS 对象 `SCENES` | 易于运营人员维护 |

### 9.2 性能目标

- 首屏渲染 < 1.5s（一般带宽）
- 拖拽 60fps（使用 `transform` + `will-change`）

### 9.3 兼容性

- Chrome 90+
- Safari 14+
- Firefox 88+
